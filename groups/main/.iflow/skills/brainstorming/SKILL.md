# Skill: Brainstorming - 头脑风暴与创意规划

## 描述
帮助用户进行结构化头脑风暴，将模糊的想法转化为具体的可执行计划。适用于产品规划、功能设计、问题解决等场景。

## 触发条件
当用户需要：
- 规划新项目或功能
- 解决复杂问题
- 寻找创意方案
- 制定 MVP 计划
- 进行可行性分析

## 执行流程

### 1. 需求澄清阶段
首先通过提问澄清以下问题：
- 目标用户是谁？
- 核心问题是什么？
- 成功的标准是什么？
- 有哪些约束条件？（时间、预算、技术栈）
- 有什么参考案例？

### 2. 创意发散阶段
使用以下技巧生成创意：
- **类比法**："这类似于什么已知事物？"
- **逆向思维**："如果反过来会怎样？"
- **组合法**："能否结合两个不同概念？"
- **极端假设**："如果没有资源限制会怎样？"

### 3. 结构化整理
将创意按以下维度分类：
```
┌─────────────────────────────────────────────────┐
│  影响高 │  快速获胜区    │  重大机遇区        │
│         │  (立即执行)    │  (重点规划)        │
├─────────┼────────────────┼────────────────────┤
│  影响低 │  填充区        │  待考虑区          │
│         │  (有空再做)    │  (需要论证)        │
└─────────┴────────────────┴────────────────────┘
         实现难度低          实现难度高
```

### 4. 输出格式
最终输出应包含：
1. **执行摘要**（3-5 句话）
2. **核心洞察**（3-5 个关键发现）
3. **推荐方案**（按优先级排序）
4. **行动计划**（下一步具体任务）
5. **风险评估**（潜在问题和缓解措施）

## 示例对话

**用户**：我想做一个帮助程序员学习的应用

**助手**：
> 让我们用 @brainstorming 来规划这个项目。首先澄清几个关键问题：
> 
> 1. **目标用户**：是初学者还是有经验的开发者？
> 2. **核心问题**：当前学习过程中最大的痛点是什么？
> 3. **差异化**：与现有产品（如 LeetCode、Coursera）有何不同？
> 4. **商业模式**：免费、订阅还是一次性付费？

## 最佳实践

### DO（应该做）
- ✅ 先充分理解问题再给出方案
- ✅ 提供多个备选方案供选择
- ✅ 考虑技术可行性和商业可行性
- ✅ 给出具体的下一步行动

### DON'T（不应该做）
- ❌ 直接跳到解决方案
- ❌ 忽视用户的约束条件
- ❌ 给出过于笼统的建议
- ❌ 只提供一种方案
