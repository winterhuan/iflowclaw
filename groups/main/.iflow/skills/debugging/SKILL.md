# Skill: Debugging - 调试专家

## 描述
系统化地诊断和解决技术问题。从问题复现到根因分析，提供结构化的调试方法论。

## 触发条件
- 程序出现 bug 或异常
- 性能问题诊断
- 生产环境故障排查
- 学习调试技巧

## 调试方法论

### 1. 问题定义阶段

#### 收集信息清单
```
□ 错误信息：完整的错误堆栈
□ 复现步骤：如何稳定复现问题？
□ 环境信息：
  - 操作系统和版本
  - 运行时版本（Node.js/Python/Go 等）
  - 依赖版本
  - 配置文件
□ 最近变更：代码、配置、基础设施的变化
□ 影响范围：多少用户/功能受影响？
□ 时间线：何时开始出现问题？
```

#### 问题分类
| 类型 | 特征 | 常见原因 |
|------|------|----------|
| 逻辑错误 | 程序运行但结果错误 | 算法错误、边界条件 |
| 运行时错误 | 程序崩溃 | 空指针、数组越界、类型错误 |
| 性能问题 | 运行缓慢 | 算法复杂度高、资源竞争、内存泄漏 |
| 并发问题 | 间歇性出现 | 竞态条件、死锁、资源竞争 |
| 环境问题 | 特定环境出现 | 配置差异、权限问题、网络问题 |

### 2. 假设生成阶段

#### 5 Whys 分析法
```
问题：服务响应慢
├─ 为什么？→ CPU 使用率高
│  ├─ 为什么？→ 某个查询占用大量 CPU
│  │  ├─ 为什么？→ 缺少索引
│  │  │  └─ 为什么？→ 表结构变更后未更新索引
│  │  │     └─ 根因：缺乏 Schema 变更流程
```

#### 常见假设模板
1. **代码层面**：
   - 最近的代码变更引入了 bug
   - 第三方库版本不兼容
   - 配置值错误

2. **基础设施层面**：
   - 资源不足（CPU/内存/磁盘）
   - 网络问题（延迟、丢包）
   - 依赖服务故障

3. **数据层面**：
   - 数据格式变化
   - 数据量激增
   - 脏数据导致处理异常

### 3. 验证阶段

#### 二分法定位
```
1. 确定问题存在的版本 A
2. 确定问题不存在的版本 B
3. 检查中间版本 C = (A+B)/2
4. 如果 C 有问题，问题在 [B, C]
   如果 C 没问题，问题在 [C, A]
5. 重复直到找到具体变更
```

#### 隔离测试
- **最小复现**：剥离无关代码，创建最小复现案例
- **环境隔离**：在干净环境中测试
- **变量控制**：一次只改变一个变量

### 4. 工具使用

#### 日志分析
```
# 关键日志点
- 请求入口和出口
- 外部调用（数据库、API）
- 状态变更
- 错误和异常

# 日志级别使用
DEBUG: 详细的调试信息
INFO: 正常流程的关键节点
WARN: 需要注意但非错误的情况
ERROR: 错误但程序继续运行
FATAL: 严重错误，程序终止
```

#### 性能分析
```bash
# Node.js
node --prof app.js
node --inspect app.js

# Python
python -m cProfile script.py
# 或使用 py-spy
py-spy top --pid <pid>

# Go
go tool pprof http://localhost:6060/debug/pprof/profile
```

#### 网络诊断
```bash
# 检查连通性
ping <host>
curl -v <url>

# 检查 DNS
nslookup <domain>
dig <domain>

# 检查连接
netstat -tlnp
ss -tlnp

# 抓包分析
tcpdump -i eth0 port 80
```

## 常见场景解决方案

### 内存泄漏排查
```
1. 确认泄漏：
   - 监控内存使用趋势
   - 确认不是正常的 GC 波动

2. 定位泄漏源：
   - Heap dump 分析
   - 查找保留大量内存的对象
   - 分析引用链

3. 常见原因：
   - 全局变量累积
   - 事件监听器未移除
   - 闭包引用外部变量
   - 缓存无限制增长
```

### 性能瓶颈分析
```
1. 确定瓶颈类型：
   - CPU 密集型：算法优化、并行化
   - I/O 密集型：异步化、缓存、批处理
   - 内存密集型：数据结构优化、流式处理

2. 分析工具：
   - Flame graph：查看热点函数
   - APM 工具：追踪请求链路
   - 数据库慢查询日志

3. 优化策略：
   - 算法优化
   - 缓存策略
   - 异步处理
   - 数据库优化（索引、查询优化）
```

### 并发问题排查
```
1. 识别症状：
   - 间歇性失败
   - 数据不一致
   - 死锁/超时

2. 诊断方法：
   - 代码审查：检查共享资源访问
   - 日志分析：查看执行顺序
   - 压力测试：提高并发度复现问题

3. 常见解决方案：
   - 锁机制（互斥锁、读写锁）
   - 原子操作
   - 无锁数据结构
   - 消息队列解耦
```

## 调试心态

### DO
- ✅ 保持冷静，系统化思考
- ✅ 从简单假设开始
- ✅ 记录每一步的尝试和结果
- ✅ 复现问题后再尝试修复
- ✅ 验证修复确实解决了问题

### DON'T
- ❌ 随机修改代码尝试修复
- ❌ 忽视错误信息
- ❌ 假设问题一定很复杂
- ❌ 不验证就部署修复
- ❌ 修复后不总结教训
