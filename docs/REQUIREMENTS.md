# iFlowClaw 设计原则

## 小到可以理解

整个代码库应该能被阅读和理解。一个 Node.js 进程，少量源文件。没有微服务，没有消息队列，没有抽象层。

## 直接调用

Agent 直接调用 iFlow SDK，无需 Docker 容器层。简单直接。

## 为一个用户构建

这不是框架或平台。它是为特定需求工作的软件。使用飞书，就支持飞书。添加实际需要的集成，不是每一个可能的集成。

## 定制 = 代码变更

没有配置蔓延。想要不同的行为，修改代码。代码库足够小，这样做是安全的。

## 核心架构

- **消息路由**：飞书渠道 → 注册群组 → 触发词 `@iFlow`
- **记忆系统**：每个群组的 `AGENTS.md` + 全局 `groups/global/AGENTS.md`
- **直接执行**：主机进程直接调用 iFlow SDK
- **扩展系统**：显式注册在 `src/extensions/index.ts`
